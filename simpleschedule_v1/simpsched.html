Simple Schedule

Simple Schedule is one of the most beautiful and intuitive apps on Google Play for managing your school or university life. You can save your classes and all tasks, from homework to exams, with ease. With Material Design its really an pleasure to use, so just try it out!
If you experience any problems or just want to request something new feel free to leave a post in the community.

Project Timetable - How it started

The main reason why i started this project was that i simply wasn't happy with the apps available at that moment. I just wanted some clean, simple but nice looking app that could help me manage my university schedule. Just right after that idea came to my mind i started drawing some elements i would like to have - especially some Material elements. This resulted in this first sketch you can see here and that's where it all started.

The first idea regarding this project was the 'Recent' section. Since this app was all about simplicity and giving one a easy and fast glance at the schedule.

The 'Recent' section

Since i knew from the beginning that this section would be the part that makes this app special (compared to normal calendars eg.) i quickly decided that this should also be the main view. There are many reasons that support this decision which i will come shortly to but first: lets talk about Material Design!

This part of the app is actually a great place too emphasize many ideas of Material Design. Since its the main interaction part of the app its also much about interaction.
Talking about used elements: Since this screen is all about dynamic information and changing content, the card view came in pretty handy.
This is because cards both are great to resemble Material's Paper metaphor but also to show varying content. So cards are nice for showing grouped content but also for single objects/information, but the thing that makes them great is the combination with actions.

Cards

So cards are good for showing grouped content but also for single information. Lets explore this with the recent cards. Basically there are two types used.
The first one is a rich media card used to show the current class. Since this is a main interaction point and the most important element it utilizes the classes bold colors to draw attention to it. It has multiple action areas, as shown in the illustration. There is a part to interact with the class - to open a more detailed view of the class. Furthermore there is navigation area - to navigate to the next class but also a support action.
As simple as the concept of a card looks, it is important to clearly structure it. Action elements, like buttons, should be clearly visible and have its one space. With using rich media it is also possible to separate the content from the actions, but it is also used as an interaction point itself.
The second card is more subtile as it isn't a main interaction point and only shows further information. Therefore it only uses gray tones. The lines clearly separate the items (classes) and these also serve as interaction point to find more detailed information.

Colors and bold imagery

Another important thing for a Material app is the usage of bold colors and imagery. These are mostly used to draw attention but also to structure the content clearly. More important though is that these help to create a pleasant experience when using an app. Therefore many hours where spent to create some nice illustrations for the classes - once again huge thanks to Aron van de Pol who helped me with creating some of those awesome illustrations.

Cloud

The ability to sync your data with the cloud and thus between different devices is a essential feature for most users. Therefore it was already added in the app's beta version. The desicion to use Google Drive for this was simply because of lack of resources to build an own sync service.

How does it work
Basically the sync service is build on the back up ability of the app. So whenever there are changes made and/or new information is added the service creates a back up of this data and pushes it to Google Drive.
The interesting part was the implementation to allow multiple devices to share the same data. Therefore every device/app that shares the same accounts adds timestamps to the changes it makes. So when one has older changes then another those doesn't get synced since the newer ones are used.

The App Icon: One of the most important things

The apps icon, as simple as it seems, is one of the most important things for an app. The reason for this is, that the icon is the first interaction point a user has with an app. Therefore many different variants were tried which resulted in this simple and clean icon. The goal was to resemble a calendar, but also showing the main style of the app. Therefore a card is used since this is the main interaction point in the apps main view.
